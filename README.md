# Сімейний бюджет — пакет для Vercel

```
api/
  parse.js        ← розбір тексту/голосу + AI-аналіз (модель OpenAI)
  data.js         ← спільне сховище бюджету (хмарна синхронізація через Vercel KV)
budget/
  index.html      ← сам застосунок, відкривається на shipherson.com/budget/
package.json
```

`api/` має лежати в **корені** проєкту (поряд із `budget/`), інакше Vercel не зареєструє функції.

---

## 1. Залити файли
Завантажте весь вміст так, щоб у корені проєкту були `api/` і `budget/` поряд.
Не кладіть `api/` всередину `budget/`.

## 2. Підключити сховище (щоб дані були спільні на всіх пристроях)
Без цього застосунок працює, але зберігає дані **лише локально в одному браузері**.

1. У проєкті на Vercel → вкладка **Storage** → **Create Database** → виберіть **KV** (Upstash Redis).
2. Назвіть, створіть, і натисніть **Connect** до цього проєкту.
   Vercel сам додасть змінні `KV_REST_API_URL` і `KV_REST_API_TOKEN`.
3. **Redeploy** проєкт (інакше функції не побачать нові змінні).

> Якщо підключаєте Upstash напряму (не через Vercel KV), функція також розуміє
> `UPSTASH_REDIS_REST_URL` / `UPSTASH_REDIS_REST_TOKEN`.

## 3. Ключ OpenAI (для вводу та AI-аналізу)
**Settings → Environment Variables:**
- `OPENAI_API_KEY` = ваш ключ `sk-proj-...`
- (необовʼязково) `OPENAI_MODEL` = напр. `gpt-4o-mini`

Після додавання — **Redeploy**.

---

## Перевірка
1. `shipherson.com/api/parse` → має показати `Method not allowed` (функція жива).
2. `shipherson.com/api/data?code=2109` → має показати `{"data":...}` або `{"data":null}` (сховище живе).
   Якщо бачите помилку про KV — сховище не підключене або не зроблено Redeploy.
3. `shipherson.com/budget/` → відкрити застосунок.
   У шапці зʼявиться іконка хмари: **зелена** = синхронізовано, **сіра** = лише локально.
4. Введіть витрату на телефоні → за кілька секунд вона зʼявиться й на компʼютері
   (інший пристрій підтягує зміни автоматично кожні ~25 секунд; іконку-хмару можна
   натиснути, щоб синхронізувати негайно).

## Як це працює
- Усі дані лежать в одному спільному записі `budget:main` у KV, захищеному кодом `2109`.
- localStorage кожного браузера лишається як миттєвий кеш і офлайн-резерв:
  якщо хмара недоступна — застосунок працює локально й синхронізується, щойно звʼязок відновиться.
- Сторінка вже налаштована звертатися до `/api/parse` і `/api/data`
  (рядок `window.BUDGET_API_URL` / `window.BUDGET_DATA_URL` у `budget/index.html`).
